# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному |Абстрактный класс |Есть общее состояние (имя, возраст) и отношение "является" (is-a). Животные — это иерархия с общей базовой структурой данных. Абстрактный класс позволяет хранить поля name и age, а метод makeSound() сделать абстрактным. |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | Интерфейс| Это отношение "умеет" (can-do). Объекты разных иерархий могут поддерживать сериализацию. Нет общего состояния, нужна множественная реализация (например, класс может одновременно быть Document и Serializable).|
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному |Абстрактный класс | Есть общее состояние (скорость, уровень топлива) и отношение "является" (is-a). Разные виды транспорта (автомобиль, самолёт, корабль) имеют общие атрибуты и поведение.|
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi |Интерфейс | Это отношение "умеет" (can-do). Объекты из совершенно разных иерархий могут иметь общую функциональность. Нет общего состояния, нужна множественная реализация.|
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс| Есть общее состояние (цвет) и отношение "является" (is-a). Все фигуры имеют общие атрибуты (цвет) и общее поведение (расчёт площади и периметра). Можно вынести общее поле color и абстрактные методы getArea() и getPerimeter().|
| Класс должен одновременно уметь летать, плавать и ездить | Интерфейсы (множественные)| Нужна множественная реализация различных способностей. Класс может реализовать интерфейсы Flyable, Swimmable, Drivable. В Java класс может наследовать только один абстрактный класс, но реализовывать множество интерфейсов.|
